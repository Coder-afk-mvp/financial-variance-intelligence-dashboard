# Financial Variance Intelligence Dashboard

A 4-week portfolio project: clean, analyze, and build an interactive dashboard for
monthly budget-vs-actual financial performance across departments, regions, and categories.

## About the data

`data/raw/financial_raw.csv` is a **synthetic dataset**, generated by `data/generate_data.py`.
It was built this way on purpose: real company budget-vs-actual data isn't publicly available,
so this script creates data with the same *kinds* of problems real finance data has —
inconsistent labels ("Marketing" / "MKT" / "marketing"), mixed date formats, missing values,
duplicate rows, currency-symbol text, negative values, and a few outliers — so the cleaning
work in Weeks 1-2 is genuine practice rather than a formality.

If you regenerate the data (`python data/generate_data.py`), note it uses a fixed random seed,
so results are reproducible.

## Analysis Validation & Reconciliation

Every number in this project must be provably correct, not just plausible. Before calling
any week "done," verify:

- [ ] `variance == actual - budget` holds for every row (use `np.isclose` for float rounding)
- [ ] Monthly department-level totals sum to the correct monthly grand total
- [ ] No duplicate `transaction_id`s remain after cleaning
- [ ] Every missing-value decision is documented (which column, what treatment, why)
- [ ] Dashboard KPI totals match the totals computed in the notebooks

This section exists because trusting data blindly is the fastest way to ship a wrong number
to a decision-maker. Document these checks explicitly in Notebook 02 (variance, duplicates,
missing-value log) and again in Week 4 (monthly reconciliation, dashboard-vs-notebook parity)
before deploying.
